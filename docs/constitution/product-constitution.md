# 產品憲法（Product Constitution）

> 本文件為產品層級之不可變原則，實作與重構時必須遵守。  
> 依據：`docs/總監操盤系統升級-規格.md`、`docs/華麗熊-總監操盤系統-最終整合版.md`。

---

## 1. 產品定位（一句話）

**以商品為中心、以利潤為目的、以 8:2 為優先序、兼具老闆／投手／行銷總監思維的操盤系統。**

系統的任務**不是呈現所有數據**，而是**幫公司做出最有價值的少數決策**。

---

## 2. 產品使命（規格最上位）

- 優先放大最值得放大的商品與素材  
- 優先止血最危險、影響最大的高預算項目  
- 不誤殺延遲型好貨與可救商品  
- 不讓真正有潛力的黑馬素材被埋沒  
- 不讓團隊浪費時間處理低影響的小事  
- 在可接受風險下，幫公司賺更多、虧更少  

---

## 3. 系統總原則

| 原則 | 內容 |
|------|------|
| **先看影響力，再看好壞** | 先處理「最爛且最有影響的」「最好且最值得放大的」。 |
| **先商品，再素材，再帳號** | 帳號是容器，商品是主角，素材是商品的槓桿。 |
| **8:2 分層落地** | 全公司層、帳號層、商品層、素材層，每一層都做 8:2。 |
| **動作比報表重要** | 輸出的是：放大、止血、觀察、不要誤殺、值得延伸、調目標成果——而不是一堆分數卡。 |
| **一天最多調整 2 次** | 系統幫你節制地調，不煽動亂調。 |

---

## 4. 介面靈魂

- 首頁第一屏只放高價值區塊，不以帳號列表為主。  
- 危險用紅、機會用綠、觀察用黃，節制用色。  
- 每一張卡都要有一句「總監判語」，不是只有數字。  
- 每個表先顯示動作與優先級，再顯示完整數據。  
- 介面符合 8:2：最重要的資訊最大、最先、最清楚。  

---

## 5. 頁面對應分層

| 頁面 | 角色 | 要點 |
|------|------|------|
| **首頁** | 今日操盤室／總監晨會桌面 | 今日最該動 5 件事、主力商品、高預算危險、黑馬素材、操作節制；帳號服務商品層。 |
| **商品作戰室** | 商品主戰場 | 總監判語、在撐/在拖的素材、下一步建議、成本比與 headroom。 |
| **素材頁** | 素材作戰台 | 所屬商品、幫/拖、黑馬/疲乏、給投手/設計一句話。 |
| **審判官** | Workflow 入口 | 五 workflow；人格不算分，只把算好的結論講成總監語言。 |

---

## 6. 與其他憲法之關係

- **決策憲法**：誰來算「真好/假好/可救/真爛」、排序、動作建議。  
- **人格憲法**：誰來把算好的結論說成人話、判語、視角。  
- **Prompt 憲法**：人格與校準的組裝順序與唯一來源。  

本產品憲法不規定具體 API 或欄位；技術實作見規格文件與 decision / persona / prompt 憲法。
