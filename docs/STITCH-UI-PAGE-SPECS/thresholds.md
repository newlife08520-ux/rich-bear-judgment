# 門檻設定

## 1. 頁面用途

- 編輯決策門檻草稿、與已發布值對照、快速套用預設、儲存草稿、發布（立即影響決策卡）、極端值警告、回滾至上版。
- 主要給管理者與會調參的投手。

## 2. 進入方式

- route：`/settings/thresholds`。
- 設定中心管理工具與 AI 作戰設定連結。

## 3. 首屏必須看到的內容

1. 標題「門檻設定」與 Sliders 圖示。
2. 「影響頁面」卡：列生命週期、成績單、汰換、審判官決策卡。
3. 「已發布 vs 草稿」對照表。
4. 「回滾至上版」按鈕區。

## 4. 區塊排列順序

- Header → 影響說明 → 對照表卡 → 條件 extremeWarnings 卡 → 快速套用預設 grid → 說明卡 → 編輯草稿卡（多 Input）→ 底部儲存草稿與發布。

## 5. 按鈕盤點

### A. 主按鈕（Primary CTA）

- 「發布（立即影響決策卡）」：與儲存草稿並列，發布為最高影響。

### B. 次按鈕（Secondary）

- 「儲存草稿」；各預設「套用」outline。

### C. 危險按鈕（Danger）

- 「回滾至上版」outline 但屬版本回退；極端值區塊警示。

### D. 輔助按鈕（Utility）

- SidebarTrigger。

## 6. 篩選 / 控制元件

- 無 tab；大量數字 Input 以百分比或數值顯示（見 FIELDS）。

## 7. 狀態資訊 / badge / warning

- 表格列 diff 琥珀底；extremeWarnings 整卡琥珀；說明文字。

## 8. 可收合但不能消失的區塊

- 已發布與草稿對照；發布按鈕與文案；影響頁面列表。

## 9. 此頁最不能被 AI 誤改的地方

- 發布與草稿分離；回滾；極端值警告；「立即影響決策卡」語意。

## 10. Stitch 用摘要

門檻設定是版本化參數頁：首屏先讀影響範圍與對照表，再改草稿。主 CTA 是發布；儲存草稿為次。設計不可把發布藏進選單或弱化極端值警告；對照表列 diff 要高對比。
