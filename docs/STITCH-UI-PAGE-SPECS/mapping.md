# 資料歸因修正

## 1. 頁面用途

- 手動將未映射的廣告活動對應到商品，並處理衝突；屬資料修正工具，非日常主流程。
- 主要給管理者與必要時的投手。

## 2. 進入方式

- route：`/mapping`。
- 側欄無連結；從設定中心「管理工具」→「資料歸因修正」進入。
- 須登入。

## 3. 首屏必須看到的內容

1. 標題「資料歸因修正」與 MapPin。
2. 說明此頁為修正工具。
3. 未映射清單標題與筆數、表格或空狀態。
4. 衝突區（若 API 有資料，見 product-mapping.tsx 後半）。

## 4. 區塊排列順序

- Header → 說明 → 未映射 Card（表）→ 衝突與操作（待確認完整檔案）。

## 5. 按鈕盤點

### A. 主按鈕（Primary CTA）

- 每列映射儲存／確認（標籤待確認表格實作）。

### B. 次按鈕（Secondary）

- 待確認是否有返回設定連結。

### C. 危險按鈕（Danger）

- 覆寫映射具資料影響；是否有二次確認待確認。

### D. 輔助按鈕（Utility）

- SidebarTrigger。

## 6. 篩選 / 控制元件

- 每列 Select 選商品（待確認）。

## 7. 狀態資訊 / badge / warning

- AlertTriangle 與未映射計數；空狀態文案。

## 8. 可收合但不能消失的區塊

- 未映射列表與衝突列表不可合併成單一摘要而無表格。

## 9. 此頁最不能被 AI 誤改的地方

- 從設定進入的路徑認知；逐列可操作表格；低頻工具語意。

## 10. Stitch 用摘要

資料歸因修正是低頻後台頁，首屏要說明目的並列出未映射與衝突。主操作在表格列上。設計應偏工具感、可掃讀，不要把表格變成裝飾卡片而難以對照 campaign 與商品。

（約 140 字）
