# 素材生命週期中心

## 1. 頁面用途

- 以七階段看板管理素材生命週期（待初審→…→靈感池），含篩選、Lucky 一鍵任務、FirstDecisionBlock 等；支援 creativeId 深連結。
- 主要給投手、營運。

## 2. 進入方式

- route：`/creative-lifecycle`。
- 側欄「更多」→ 素材生命週期。
- URL 參數 creativeId 待確認高亮行為（見檔頭註解）。

## 3. 首屏必須看到的內容

1. 標題「素材生命週期中心 1.0」與七階段一句話說明。
2. 卡片標題「素材清單」與右側篩選：Lucky 一鍵按鈕（條件顯示）、階段 Select、ROI label Select。
3. 載入中或空狀態（含前往素材中心／投放中心／設定中心連結）或欄位看板。

## 4. 區塊排列順序

- Header → 主 Card（標題列＋篩選）→ LifecycleColumn 等內容區 → 其餘區塊見 CreativeLifecyclePageView 後半（待完整掃描）。

## 5. 按鈕盤點

### A. 主按鈕（Primary CTA）

- Lucky 一鍵生成補量任務（條件顯示，secondary）。

### B. 次按鈕（Secondary）

- 空狀態內 outline 連結：前往素材中心、投放中心、設定中心。

### C. 危險按鈕（Danger）

- 待確認是否有批次汰換或刪除。

### D. 輔助按鈕（Utility）

- Filter 圖示旁 Select；SidebarTrigger。

## 6. 篩選 / 控制元件

- 階段與 ROI label 兩個 Select 應在清單標題列同排。

## 7. 狀態資訊 / badge / warning

- 空狀態虛線邊框說明門檻；載入中文案。

## 8. 可收合但不能消失的區塊

- 七階段語意與空狀態導流連結不可刪；Lucky 按鈕規則保留。

## 9. 此頁最不能被 AI 誤改的地方

- 階段定義文案；空狀態三連結；與門檻設定頁的決策關聯（見門檻設定影響頁面列表）。

## 10. Stitch 用摘要

素材生命週期是七階段營運看板，首屏要讀懂階段定義與篩選。主操作含 Lucky 一鍵任務（有資料時）。空狀態必須引導到素材、投放、設定三處而非空白。視覺優化應保留階段欄位結構，勿改成單時間軸而遺失營運語意。

（約 180 字）
