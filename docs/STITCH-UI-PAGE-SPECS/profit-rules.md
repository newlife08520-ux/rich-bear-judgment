# 獲利規則中心

## 1. 頁面用途

- 每商品設定成本比、目標淨利率、樣本門檻；顯示保本 ROAS 與目標 ROAS；欄位變更時呼叫 API 儲存。
- 主要給管理者與投手。

## 2. 進入方式

- route：`/settings/profit-rules`。
- 側欄或設定中心連結。

## 3. 首屏必須看到的內容

1. 標題與公式說明卡。
2. 新增商品名稱輸入。
3. 商品規則卡片與試算數字。

## 4. 區塊排列順序

- Header → 公式卡 → 新增列 → 多張商品 Card。

## 5. 按鈕盤點

### A. 主按鈕（Primary CTA）

- 無全域儲存鈕；以輸入即存為主流程。

### B. 次按鈕（Secondary）

- 無。

### C. 危險按鈕（Danger）

- 無獨立按鈕；錯誤參數導致 ROAS 無法計算時顯示為橫線。

### D. 輔助按鈕（Utility）

- SidebarTrigger。

## 6. 篩選 / 控制元件

- 無 tab。

## 7. 狀態資訊 / badge / warning

- Toast 成功或失敗。

## 8. 可收合但不能消失的區塊

- 公式說明；每卡試算兩行。

## 9. 此頁最不能被 AI 誤改的地方

- 即存行為需讓使用者理解；公式不可刪。

## 10. Stitch 用摘要

獲利規則是每商品一卡加頂部公式。沒有單一儲存按鈕，改數字即寫入。設計要讓試算與輸入同屏。勿把教育文字藏太深。

（約 110 字）
