# 投放紀錄

## 1. 頁面用途

- 唯讀列表顯示投放操作紀錄（時間、訊息、狀態、草稿 ID），最新在前。
- 主要給投手、營運稽核。

## 2. 進入方式

- route：`/publish/history`。
- 側欄更多 → 投放紀錄。

## 3. 首屏必須看到的內容

1. SidebarTrigger、標題「投放紀錄」。
2. 載入中／錯誤／空狀態／表格四態之一。

## 4. 區塊排列順序

- Header → 單一 Card 包 Table 或狀態內容（max-w-4xl 閱讀寬）。

## 5. 按鈕盤點

### A. 主按鈕（Primary CTA）

- 無（唯讀頁）。

### B. 次按鈕（Secondary）

- 無。

### C. 危險按鈕（Danger）

- 無。

### D. 輔助按鈕（Utility）

- SidebarTrigger。

## 6. 篩選 / 控制元件

- 無搜尋或篩選於本檔。

## 7. 狀態資訊 / badge / warning

- Badge variant secondary 顯示 status；錯誤與空狀態完整文案。

## 8. 可收合但不能消失的區塊

- 四欄表頭：時間、訊息、狀態、草稿 ID。

## 9. 此頁最不能被 AI 誤改的地方

- 草稿 ID 單欄字體 mono 可讀性；錯誤狀態不可靜默。

## 10. Stitch 用摘要

投放紀錄是簡單唯讀表。首屏標題下即狀態或表格，四欄結構固定。無主 CTA。設計應保持可掃時間與狀態 badge，勿為極簡而刪除草稿 ID 欄。

（約 120 字）
