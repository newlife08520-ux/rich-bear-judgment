# 行動紀錄

## 1. 頁面用途

- 管理行銷任務：優先級、截止日、影響金額、狀態、負責人；批次改狀態與指派、複製今日執行清單、建立任務。
- 主要給營運、投手、管理者。
- 可搭配 URL 參數 highlight 任務（見 tasks.tsx 註解）。

## 2. 進入方式

- route：`/tasks`。
- 側欄「更多」→ 行動紀錄。
- 須登入；API 錯時顯示診斷卡片。

## 3. 首屏必須看到的內容

1. 頁首：SidebarTrigger、標題「行動紀錄」。
2. 按鈕「複製為今日執行清單」、「建立任務」。
3. 三張卡：高優先、今日到期、高影響金額。
4. 摘要列：待分配／進行中／完成數字與「只看我負責」勾選。
5. TasksDataTable 表頭或載入中。

## 4. 區塊排列順序

- Header（左標題、右兩按鈕）→ 三卡 grid → 摘要 Card → 條件錯誤卡 → 條件批次列 → 表格 → TasksDialogs。

## 5. 按鈕盤點

### A. 主按鈕（Primary CTA）

- 「建立任務」：頁首右側。

### B. 次按鈕（Secondary）

- 「複製為今日執行清單」；批次改狀態、批次指派（有勾選時）。

### C. 危險按鈕（Danger）

- 批次操作影響多筆；實際確認在 TasksDialogs 內（待確認文案）。

### D. 輔助按鈕（Utility）

- 「取消選取」、SidebarTrigger。

## 6. 篩選 / 控制元件

- Checkbox「只看我負責」留在摘要列；表格內篩選排序見 TasksDataTable（待確認）。

## 7. 狀態資訊 / badge / warning

- 「任務資料暫不可用」琥珀區塊含 migration 提示，不可改成小 toast。
- 只看我負責時的 dev 提示文字（模擬身份）不可刪。

## 8. 可收合但不能消失的區塊

- 三張 KPI 卡、摘要列、條件式批次列規則、錯誤診斷區。

## 9. 此頁最不能被 AI 誤改的地方

- 建立任務與複製清單同列首屏；批次與勾選；錯誤狀態完整呈現。

## 10. Stitch 用摘要

行動紀錄首屏要呈現今日壓力與隊列狀態，右上固定「複製今日執行清單」與「建立任務」。主 CTA 是建立任務。表格與批次操作不可被純卡片流取代；API 失敗要有明顯診斷區。視覺應強調已選筆數與批次後果。
