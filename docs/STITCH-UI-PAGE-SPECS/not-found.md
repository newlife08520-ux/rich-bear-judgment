# 404 Page Not Found

## 1. 頁面用途

- 告知使用者目前 URL 無對應前端路由（wouter 預設 Route）。
- 給全角色。

## 2. 進入方式

- 任意未匹配 path（登入狀態下仍會進 main 內 Switch 的 fallback）。
- 無側欄導航輔助（整頁置中卡）。

## 3. 首屏必須看到的內容

1. AlertCircle 與「404 Page Not Found」標題（英文）。
2. 說明文字「Did you forget to add the page to the router?」。

## 4. 區塊排列順序

- 全屏灰底置中單一 Card。

## 5. 按鈕盤點

### A. 主按鈕（Primary CTA）

- 無。

### B. 次按鈕（Secondary）

- 無。

### C. 危險按鈕（Danger）

- 無。

### D. 輔助按鈕（Utility）

- 無返回首頁按鈕於目前檔案（**待確認**是否應產品補上）。

## 6. 篩選 / 控制元件

- 無。

## 7. 狀態資訊 / badge / warning

- 僅錯誤語意；無額外 badge。

## 8. 可收合但不能消失的區塊

- 404 訊息本體；若補「回首頁」連結應固定可見。

## 9. 此頁最不能被 AI 誤改的地方

- 勿改成空白或自動導向而不告知；與開發者英文文案若產品要中文化需另決策。

## 10. Stitch 用摘要

404 為全屏單卡，英文標題與開發者提示。無按鈕、無側欄。Stitch 若要優化應加回首頁／返回導航並中文化，但需產品決策；目前規格以程式為準勿擅自刪除錯誤提示。

（約 130 字）
