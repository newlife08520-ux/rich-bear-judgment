## 4. V14 被拆成哪些工作流招式

- **create**：V14 的「直接可用的字幕文案、爆款剪輯方案、銷售頁架構」拆入。Overlay 要求：直接產出、不先打分；文案／腳本給主標／副標／建議字幕；影片給 0–3 秒／中段／結尾畫面與字幕、剪輯邏輯、爆款外掛；銷售頁給各屏重點、說服順序、GIF/視覺補強腳本、遺漏細節、AOV 拉升。
- **audit**：V14 模式 A（素材煉金術）與模式 B（轉單說服力）的審核標準與輸出骨架拆入。Overlay 要求：才顯示分數／通過與否／阻擋原因／待補項；審素材時走產品定位→第一眼生死判決→鉤子痛點→實質字幕/剪輯建議→剪輯醫生→爆款外掛→總監總結與修改優先級；審銷售頁時走預判死亡點→地獄級評分→概念升級→文案優化→GIF 腳本→遺漏細節→客單價策略→總監總結。
- **clarify**：V14 未直接對應，依需求撰寫：不亂審、先追問或陪跑、自然不油。
- **strategy**：V14 未直接對應，依需求撰寫：排序、優先級、建議動作、原因、why not more。
- **task**：V14 未直接對應，依需求撰寫：任務列表、標題、執行說明、理由。

## 5. 五個工作流各自的用途與輸出規則

| 工作流 | 用途 | 輸出規則 |
|--------|------|----------|
| **clarify** | 資料不足、問題模糊、陪跑收斂 | 不亂審、不亂給假專業意見；先追問或陪跑；可自然、有一點人味，不油；不主動出評分卡 |
| **create** | 寫文案、腳本、銷售頁架構、延伸方向 | 直接產出內容；不先打分；保留可直接拿去改的執行感（字幕／主標／架構／剪輯建議／GIF 腳本等）；資訊不足可簡短追問 |
| **audit** | 真正要審素材／頁面／文案／內容 | 才顯示分數、通過與否、阻擋原因、待補項；輸出含結構化評分卡（JSON）；審素材走 V14 模式 A 骨架，審頁面走模式 B 骨架 |
| **strategy** | 投放、資源分配、先救誰、該拉該停、8:2 | 輸出排序、優先級、建議動作、原因、why not more；條列或短段；不輸出評分卡；資料不足可先問 |
| **task** | 拆給設計／投手／企劃 | 轉成任務列表：任務標題、執行說明、理由；簡潔可執行；不是評論、不是審核 |

## 6. Prompt 最終組裝順序

1. **Core Persona**（唯一人格真源，`rich-bear-core.ts`）
2. **Published Overlay**（可選，workbench 主 prompt，不取代人格）
3. **Hidden Calibration**（隱性校準層，`getHiddenCalibration()` 整份）
4. **Workflow Overlay**（`getWorkflowOverlay(workflow)`；若 workflow=audit 再疊 MODE A/B/C/D 焦點）
5. **Data Context**（本任務資料，由呼叫方帶入）
6. **Output Schema**（僅當 workflow=audit 時附加結構化評分卡）

## 7. 目前哪些地方仍是假模式切換（本輪未修 UI）

- **QUICK_PROMPTS 四顆按鈕**（幫我看素材、產出銷售頁架構、發想痛點短影音、找出文案盲點）：仍只呼叫 `handleQuickPrompt(text)`，只把固定句塞進輸入框，**沒有**設定 `workflow`。使用者若未手動選工作流，會依後端意圖推論，可能正確；但按鈕本身未「真正切換工作流」。
- **側欄「常用審判模式」三顆**（Boss／投手／創意）：改的是 **uiMode**（persona 偏向），不是 **workflow**。若使用者以為在選「審判」而沒選「幫我審」（audit），後端仍可能因意圖推論進 audit，但 UI 未明確標示目前是 clarify/create/audit/strategy/task 哪一個，容易混淆。
- **空狀態四大卡片**：已改為同時設定 `workflow` + `uiMode` + 預設 prompt，為**真工作流切換**。
- **輸入區上方五顆工作流按鈕**：已為真切換（`setWorkflow(w)`，送出時 body 帶 `workflow`）。

建議下一輪：讓 QUICK_PROMPTS 對應到 workflow（例如「幫我看素材」→ setWorkflow(audit) + prefill），並在側欄或標題處標示目前工作流，減少假模式感。
