# 華麗熊結構收斂 1.0 — 驗收結果與 Overlay 結構化欄位規劃

## 1. Staging 驗收結果

### 1.1 自動腳本驗收（已執行）

- **Overlay 驗證邏輯**（`npx tsx script/verify-overlay-validation.ts`）  
  - 正常視角補充（Boss / 投手）：**通過**，`validateOverlayContent` 回傳 ok=true。  
  - 人格級內容（你的最高任務、角色核心、分數哲學、Hidden Calibration）：**阻擋**，ok=false，matchedLabel 正確。  
  - **結果：6 通過、0 失敗。**

### 1.2 建議手動驗收清單（Staging 實機）

以下需在 staging 環境手動操作一次：

| 項目 | 操作 | 預期結果 |
|------|------|----------|
| Overlay 頁正常視角補充 | 進入「角色視角 Overlay」→ 選 Boss 視角 → 草稿輸入「此視角下優先顯示風險與金額；建議動作列在第一段。」→ 儲存草稿 → 發布 | 儲存與發布成功，無錯誤 |
| Overlay 頁人格級阻擋 | 同一頁草稿改為「你的最高任務是讓產品更會賣」→ 儲存或發布 | 後端回 400，畫面顯示錯誤訊息（此區僅能填寫視角補充…），未寫入／未發布 |
| 單次審判輸入不足 | 呼叫 POST /api/content-judgment/start，body 僅含極短 text（如 "幫我審" 少於 30 字）且無 url | 回 400，message 為「總監需要一點素材才能下判：請貼上連結、或至少 30 字以上的文案／說明，我再幫你審。」 |
| Chat audit 輸入不足 | 審判頁選「幫我審」、不貼連結不上傳、輸入少於 30 字 → 送出 | 回 200，助理訊息「請先上傳素材、貼上連結或貼上要審的文案／數據，我再幫你審。」（needMoreInput） |
| 單次審判與 chat audit 體感一致 | 同一份素材（例如一則 30 字以上文案或一則 URL）分別用單次審判 API 與 chat 選「幫我審」送出 | 兩邊皆產出評分卡格式、皆有 MODE 焦點與結構化輸出，體感一致 |

## 2. 哪些提示仍太像系統、不像華麗熊

- **已調整**  
  - 單次審判輸入不足：由「請提供 URL 或至少 30 字以上內容再審。單次審判與對話審判一致…」改為「**總監需要一點素材才能下判：請貼上連結、或至少 30 字以上的文案／說明，我再幫你審。**」與 chat 的「我再幫你審」一致，較像總監口吻。

- **仍偏系統感、可之後微調**  
  - Overlay 防呆回傳的 `reason`：目前為「此區僅能填寫『視角補充』與『輸出偏向』，不可重寫人格或校準層。偵測到不當內容：xxx。請移除…」— 清楚但偏說明文，若未來要更有人味可改為一句總監式提醒再補說明。  
  - 前端按鈕／錯誤區若日後改版，可統一為「總監說：xxx」式文案。

## 3. Overlay 結構化欄位建議

目標：未來以結構化欄位取代大段自由文字，避免再次形成第二人格入口。以下為**規劃**，尚未實作。

### 3.1 Boss 視角

| 欄位概念 | 說明 | 型別建議 |
|----------|------|----------|
| 摘要優先順序 | 先排風險／金額／建議動作／結論 | 多選或順序列舉 |
| 摘要長度 | 簡短（1–3 行）／中等／完整 | enum |
| 商業結論風格 | 先給結論／先給證據／並重 | enum |
| 是否先顯示風險 | 是／否 | boolean |
| 是否先顯示金額／影響範圍 | 是／否 | boolean |
| 建議動作出現位置 | 第一段／與結論並列／獨立區塊 | enum |

### 3.2 投手視角

| 欄位概念 | 說明 | 型別建議 |
|----------|------|----------|
| 預設展開區塊 | rescue / scale up / no misjudge / extend 等哪些預設展開 | 多選 |
| 排序偏好 | 依 ROAS／CVR／花費／轉換數／素材健康度等 | 多選或優先順序 |
| 優先層級 | campaign / product / creative 哪一層先看 | enum |
| whyNotMore 呈現方式 | 簡短一句／獨立段／與建議動作合併 | enum |
| 哪張表先看 | 依實際產品表單命名列舉 | enum 或 key |

### 3.3 創意視角

| 欄位概念 | 說明 | 型別建議 |
|----------|------|----------|
| 先看維度 | 鉤子／前 3 秒／首圖／字幕／CTA 等 | 多選或順序 |
| 產出形式 | 先給 3 方向／1 完整版／並行 | enum |
| 靈感池偏向 | 延伸方向／設計借鑑／爆款拆解 等 | 多選 |
| 輸出形式偏向 | 創作（直接給文案腳本）／改稿建議（先點問題再給改法） | enum |

### 3.4 共用或備選

- 各視角可共用：**語氣微調**（更簡短／更詳盡）、**數字呈現**（要不要帶單位與區間）。  
- 實作時可先做「必選 + 少量選項」，再依使用情況擴充；仍保留一欄「自由補充」（字數上限與防呆），避免又變第二人格。
