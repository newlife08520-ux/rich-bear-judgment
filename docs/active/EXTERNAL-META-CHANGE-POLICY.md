# 外部 Meta 變更政策

## 偵測到漂移時

- **UI**：顯示「原生後台可能已變更」；AI／規則產出應視為 **可能 stale**，先 **更新資料** 再採高風險動作。
- **節奏**：使用者可點「已讀並校準今日調整節奏」重置今日 `adjustCount`（見 `resetAdjustCountsForUserToday`）。

## 未偵測到時

- 不保證後台無變更；僅表示與**上次 ingest 快照**比對之一致性。

## 與 execution 的邊界

- 經本系統 **apply** 成功之變更，會隨下次 refresh 進入新快照；與「外部偷改」區分依賴比對時序與營運流程。
