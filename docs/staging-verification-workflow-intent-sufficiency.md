# Staging 驗收：工作流切換、意圖判讀、輸入充足度

## 1. 工作流偏向按鈕（真切換）

- 到審判頁 `/judgment`，在輸入區上方應看到「工作流：陪跑收斂、幫我做、幫我審、看策略、轉任務」五個按鈕與「目前：xxx」標籤。
- 點「幫我審」後再送出一則**有附件或貼連結或 30 字以上**的訊息：回覆應為評分卡格式（含 score、一句總判決、先做什麼等）。
- 點「幫我做」後送出「幫我寫一段 FB 貼文」：回覆應為直接產出文案，**不要**先出現評分卡。
- 點「陪跑收斂」後送出一般問題（如「這週預算怎麼抓？」）：回覆應為自然對談、可追問，**不要**強制評分卡。
- 點「看策略」後問「哪個該拉、哪個該停」：回覆應為優先順序與建議動作，不要評分卡。
- 點「轉任務」後問「幫我拆成任務給團隊」：回覆應為任務列表，不要評分卡。

## 2. 意圖自動判讀（不按鈕時）

- 工作流維持「陪跑對談」，輸入「幫我看這支素材」或「審一下這篇文案」：後端應推斷為 judge，回覆為評分卡（若有足夠輸入）。
- 輸入「幫我產出銷售頁架構」：應推斷為 create，回覆為架構產出，非評分卡。
- 輸入「哪個該拉、哪個該停」：應推斷為 strategy。
- 輸入「拆成任務給團隊」：應推斷為 task。

## 3. 輸入充足度（審判時才檢查）

- 工作流為「幫我審」（或意圖推斷為 audit），輸入僅「幫我審」且**無附件、無 URL、字數 < 30**：應回 200，內容為助理追問「請先上傳素材、貼上連結或貼上要審的文案／數據，我再幫你審。」，且 **不呼叫 AI**（needMoreInput 路徑）。
- 同一則若改為貼上一段 30 字以上文案或一個連結：應正常呼叫 AI 並回傳評分卡。

## 4. 空狀態入口與側欄

- 空狀態四大卡片：點「素材審判」應設 workflow=audit、uiMode=創意；點「商品頁架構」應設 workflow=create、uiMode=boss。
- 側欄「常用審判模式」仍為 Boss/投手/創意（uiMode），不與工作流混淆；送出時 body 同時帶 `uiMode` 與 `workflow`。

## 5. 新對話重置

- 點「新對話」後，工作流應重置為「陪跑收斂」。

## 6. API 行為摘要

- `POST /api/content-judgment/chat` body 可選 `workflow: "clarify"|"create"|"audit"|"strategy"|"task"`。
- 未帶 `workflow` 時，後端依 `message.content` 關鍵字推斷。
- 僅當 effectiveWorkflow === "audit" 且輸入不足時回傳 `needMoreInput: true` 與追問文，不呼叫 Gemini。
- 回傳 JSON 含 `workflow`（本次使用的工作流），前端可據此更新 UI。
