# 素材中心流程定義

本文件定義**使用流程與欄位責任**，供 Phase B 實作依循。不涉及投放中心、左側主架構與分析區。

---

## 1. 使用者主流程

### 步驟說明

| 步驟 | 使用者做什麼 | 說明 |
|------|--------------|------|
| **第一步** | 建立素材包（或選既有素材包） | 必填僅「名稱」；產品名稱預設自動帶入名稱、可編輯、可後補；可立刻儲存，不填文案也能建立。 |
| **第二步** | 上傳檔案並建立版本 | **建立素材包成功後，自動選中該素材包並展開/聚焦版本區**，使用者可立刻開始上傳圖片/影片；選類型、比例，上傳或貼 URL，填檔名（可自動帶入），必要時設主版本 → 儲存版本。 |
| **第三步** | （可後補）補預設文案、標題、CTA、網址、備註 | 在素材包編輯的文案區填寫；文案區預設摺疊，不擋建立與上傳。 |

- **上傳發生在第幾步**：第二步（選定素材包之後；新建成功後自動進入上傳情境）。  
- **文案補寫發生在第幾步**：第三步；文案區預設摺疊，區塊標題明確標註「預設文案，可後補」，一進素材中心不要先看到一堆文字欄位。  

### 建立素材包後：直接進入上傳模式

- 建立素材包**成功後**：系統**自動選中該素材包**、**自動展開/聚焦到版本區**。  
- 使用者可**立刻**開始上傳圖片/影片版本，不需再自己找按鈕或區塊。  

### 完成一個素材包的最短路徑

1. 點「建立素材包」→ 只填「名稱」（產品名稱預設帶入名稱，可改可留）→ 儲存。  
2. 系統自動選中該包並展開版本區 → 上傳一檔 → 選類型與比例 → 儲存版本。  
3. 結束。（文案、標題、CTA、網址、備註皆可之後再補；廣告目的、狀態 Phase B 先不顯示在主流程。）

---

## 2. 最低必填

### 建立素材包時

- **名稱**：必填。  
- **產品名稱**：**非必填**。預設自動帶入與「名稱」相同內容，可編輯、可後補；使用者只打名稱也能建立。  

其餘欄位（主文案、標題、CTA、落地頁 URL、備註、廣告目的、狀態）**皆非建立時必填**。  

Phase B 依此實作：僅名稱必填，產品名稱預設帶入名稱並可後補。

---

## 3. 欄位責任分類

| 欄位 | 分類 | 說明 |
|------|------|------|
| **名稱** | 建立時必要 | 素材包辨識用，建立當下必填。 |
| **產品名稱** | 建立後可補 | 預設自動帶入名稱，可編輯、可後補；取代「品牌/產品」單一欄位。 |
| **主文案** | 建立後可補 | 預設創意用，可留空建立；屬文案區，預設摺疊，標註「可後補」。 |
| **標題** | 建立後可補 | 同上。 |
| **CTA** | 建立後可補 | 同上；系統可預設「來去逛逛」，仍可後改。 |
| **落地頁 URL** | 建立後可補 | 同上。 |
| **備註** | 建立後可補 | 選填，多數可留空。 |
| **廣告目的** | 進階設定／Phase B 先隱藏 | 預設「銷售」；**Phase B 不在主流程顯示**，必要時再放進進階或之後階段。 |
| **狀態** | 進階設定／Phase B 先隱藏 | 預設「草稿」；**Phase B 不在主流程顯示**，必要時再放進進階或之後階段。 |

- **建立時必要**：只有「名稱」。  
- **建立後可補**：產品名稱（預設帶入名稱）、主文案、標題、CTA、落地頁 URL、備註。  
- **進階／Phase B 先隱藏**：廣告目的、狀態（後端預設「銷售」「草稿」，UI 先不顯示在主畫面）。

---

## 4. 上傳後的行為

### 上傳完一個版本後，畫面會看到什麼

- 上傳 API 回傳 `fileUrl`、`fileName`、`fileType`、`storageProvider`，表單會帶入檔名與類型。  
- 使用者需選「類型（圖/影片）」「比例」並儲存版本，該筆版本才會寫入素材包。  
- 儲存後：該素材包的版本列表會多一筆，可看到檔名、類型、比例、主版本標示；若有縮圖則顯示縮圖。

### 使用者下一步通常會做什麼

- **只上傳多個版本**：繼續上傳、選類型比例、儲存，直到版本齊全。  
- **或補文案**：到素材包編輯，填主文案、標題、CTA、網址等。  
- **或到投放中心**：選此素材包建立投放。

### 是否可以「先只上傳版本，之後再補文案」

- **可以**。  
- 建立素材包時只需名稱與產品名稱；建立版本時只需類型、比例與檔案（及檔名）。  
- 主文案、標題、CTA、網址、備註一律可後補，不擋建立、不擋上傳。

---

## 5. UI 目標

### 素材中心這一頁不應該像什麼

- 不應該像**建立廣告表單**（一進去就一堆投放/創意欄位必填）。  
- 不應該像**一次要把所有欄位填完才能存**的流程。  
- 不應該讓**廣告目的、狀態**擋在建立或上傳之前。

### 素材中心這一頁應該像什麼

- 應該像**快速整理素材與版本的工作台**：先建包、先上傳、先分好版本，再慢慢補文案。  
- 應該讓**上傳與版本**明顯、優先；**文案區（主文案/標題/CTA/落地頁/備註）為次要區塊，預設摺疊**，區塊標題或說明清楚寫「預設文案，可後補」，**不要一進素材中心就看到一堆文字欄位**。  
- 應該讓**完成一個素材包的最短路徑**就是：只填名稱 → 儲存 → 自動進入版本區 → 上傳至少一版 → 完成。  
- **廣告目的、狀態**：Phase B 先不顯示在主流程，後端預設「銷售」「草稿」；主畫面保持乾淨，必要時再放進進階或之後階段。

## 6. Phase B 實作邊界

### Phase B 會改哪些檔案

- **後端**：`server/modules/asset/asset-package.schema.ts`（建立時僅名稱必填；產品名稱選填、可預設帶入名稱；廣告目的、狀態預設「銷售」「草稿」；文案類可空）。  
- **前端**：`client/src/pages/assets.tsx`（表單順序、區塊主次；**建立成功後自動選中該包並展開版本區**；**文案區預設摺疊、標註「預設文案，可後補」**；產品名稱預設帶入名稱、可編輯；**廣告目的/狀態 Phase B 先不顯示在主流程**；不擋建立與上傳）。

### Phase B 不會改哪些檔案

- **左側與路由**：`App.tsx`、`client/src/components/app-sidebar.tsx`。  
- **分析與設定**：`dashboard.tsx`、`fb-ads.tsx`、`ga4-analysis.tsx`、`judgment.tsx`、`history.tsx`、`settings.tsx`、`login.tsx`、`not-found.tsx`。  
- **投放**：`publish-placeholder.tsx`、`publish-history-placeholder.tsx`、`server/modules/publish/*`。  
- **上傳與版本**：Phase A 已完成的 provider、storageProvider、version API、upload API 不變；`asset-version-repository`、`asset-version-service`、`asset-package-repository`、`asset-package-service` 介面與資料結構不變（僅 schema 放寬必填/預設）。  
- **共用**：`shared/schema.ts` 僅在必要時為 AssetPackage 放寬型別或維持現狀，不動其他模組。

### 哪些既有行為保證不變

- 左側導覽、戰情、FB、GA、判讀、紀錄、設定：無任何改動。  
- 投放中心與投放紀錄：無任何改動。  
- 素材中心對外 API 路徑與回應格式不變；已存在素材包與版本仍可正常編輯、顯示、刪除。  
- 登入與權限、userId 隔離不變。
