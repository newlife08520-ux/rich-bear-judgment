# P0 資訊架構：廣告決策工作台（Product-First / Action-First / Owner-First）

## 核心原則

- **Product-first**：主視角為「商品」，不是帳號；判斷回到「哪個商品加碼/停損、哪些素材保留/續測/關閉/重製、誰負責」。
- **Action-first**：先給結論與待辦，再給原因；每頁定義「給誰看、看完要做什麼」。
- **Owner-first**：商品/投手/素材/任務皆有 owner 或 assignee，狀態可追蹤（未指派→已指派→進行中→已完成→待確認）。

## 角色模式

| 模式 | 主要看什麼 | 資訊密度 | 出口 |
|------|------------|----------|------|
| 老闆 | 整體、商品勝負、責任、今日決策、風險 | 少雜訊、多結果 | 今日 3 件事、誰該處理 |
| 投手 | 今天要調整什麼：動作、幅度、理由、優先級 | 任務導向 | 回 Ads Manager 做什麼 |
| 素材策略 | 素材表現、疲勞、翻拍方向、商品對應 | 素材維度 | keep/test/kill/remake、下一波方向 |

## 頁面分工

1. **今日決策中心（首頁 /**）**  
   30 秒內掌握：今日 KPI、加碼 Top3、危險 Top3、待處理素材 Top5、高優先警報、待分配任務；角色切換（老闆/投手/素材）。

2. **商品作戰室（/products）**  
   主列表單位＝商品。每列：名稱、商品/投手/素材 owner、花費/營收/ROAS、CTR/CVR/CPC/CPA、啟用/勝出/疲勞素材數、狀態、AI 建議、下一步、指派狀態、最後更新。詳情 Tabs（P1）：概覽/素材/廣告組/帳號分布/AI/紀錄。

3. **素材作戰室（P1）**  
   可執行列表：狀態 保留/續測/關閉/重製/疲勞/勝出/需補新素材；篩選與負責人。

4. **投手工作台（P1）**  
   今日任務：建議動作、幅度、理由、優先級、assignee、狀態、執行清單。

5. **風險與警報中心（P1）**  
   Triage：高/中/低風險、影響商品、owner、狀態、建議處理。

6. **AI 判讀中心（/judgment）**  
   左：待判決/快速入口/Saved views；中：AI 決策卡（今日總結、立即處理、商品判決、素材判決、預算建議、owner 建議、不確定因素、可複製執行清單）；右：證據、指標、理由、歷史。非聊天室，為策略面板。

## 狀態系統

- **商品**：加碼 / 觀察 / 危險 / 停損  
- **素材**：保留 / 續測 / 關閉 / 重製 / 疲勞 / 勝出 / 需補新素材  
- **任務**：未指派 / 已指派 / 進行中 / 已完成 / 待確認  

## 規則層（前端可運作）

- 花費達門檻 + ROAS 低於目標 → 停損候選  
- CTR 高 + CVR 低 → 疑頁面/疑受眾  
- CTR 低 + CPC 高 + 花費已大 → 素材問題  
- 頻率高 + CTR 下滑 → 疲勞候選  
- 新素材資料不足 → 續測  
- 同商品明顯高表現素材 → winner  
- 同商品多支素材都燒錢 → 需補新素材  

## Filter Bar + Saved Views

- **Filter**：日期、商品、負責人、狀態、優先級、最低花費門檻、Sort。  
- **Saved Views**：老闆晨會、投手今日待辦、高潛力加碼、低 ROAS 停損、素材疲勞汰換、新素材續測。  

## Owner 機制

- 標的：商品 owner、投手 owner、素材 owner、任務 assignee、最後更新者、最後處理時間。  
- 狀態：未指派 / 已指派 / 進行中 / 已完成 / 待確認。
