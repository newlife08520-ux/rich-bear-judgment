## 1. 這一階段到底解決了什麼痛點

- **CTA 不再手打、預設一致**：改為下拉選單，預設「來去逛逛」；未填或空白送出時自動帶入「來去逛逛」，減少重複輸入與錯誤。

## 4. 目前使用者操作流程變成什麼

1. 進入投放中心 → 點「建立投放草稿」。
2. **投放設定**：廣告帳號用可搜尋下拉選（輸入關鍵字篩選）；粉專/IG 為選填輸入，placeholder 說明之後可從下拉選；其餘欄位同前（目標、名稱、預算、受眾、placement、排程）。
3. **選素材包**：下拉選一個包，自動帶入主文案、標題、CTA、網址等；CTA 可改為下拉內其他選項或沿用素材包值。
4. **選素材版本**：列表顯示縮圖/影片封面或圖示、比例、主版本；勾選一筆或多筆（可只選一個尺寸）；選中列有 ring 與背景色、可點整列勾選。
5. **覆寫文案**：選填；預設沿用素材包；CTA 為下拉、預設「來去逛逛」；送出時未填 CTA 則以「來去逛逛」送出。
6. **進階選項**（摺疊）：可改狀態，新建預設為草稿。
7. 送出 → 建立/更新草稿；列表顯示草稿與狀態。

## A. 如何承接 C-2 的現況

### 目前素材中心哪些能力可直接給投放中心使用

- **素材包列表與選取**：GET 素材包、選一個包後帶入主文案/標題/CTA/網址，投放中心直接沿用。
- **版本列表 API**：依 `assetPackageId` 取得該包版本，投放中心用於「選素材版本」。
- **版本欄位**：`assetType`、`aspectRatio`、`fileUrl`、`thumbnailUrl`、`fileName`、`isPrimary` 等，投放中心用於顯示縮圖/封面、比例、主版本標籤。
- **單一或多版本勾選**：允許只勾一筆或勾多筆，不強制多尺寸，投放中心未加額外限制。

### 哪些限制先接受

- 影片比例在素材中心為「待確認」或手選，投放中心顯示的比例即為該欄位值，不在本輪補影片 metadata。
- 素材中心無批次移動/標籤，投放中心不依賴這些功能。
- 左側素材包列表未虛擬化、C-2 有版本實機畫面未補驗，不影響本輪投放表單與版本選擇。

### 哪些限制沒有在本輪補

- 影片比例自動判斷（仍為素材中心/後端議題）。
- 批次移動、批次標記。
- 粉專/IG 從 API 取得清單並改為可搜尋下拉（僅預留 placeholder，未接 API）。
- 素材中心左側虛擬化與 C-2 實機畫面驗收。

## B. 這一輪有沒有真的比 Meta 後台更快（投手流程）

### 建立一筆草稿要幾步

1. 點「建立投放草稿」  
2. 選廣告帳號（可搜尋下拉，一步）  
3. 選粉專/IG（選填，多數可略過）  
4. 填 Campaign/Ad Set/Ad 名稱、預算、受眾、placement、排程（與 Meta 後台類似，但受眾/placement 為策略選項，略簡化）  
5. 選素材包（一下拉）→ 自動帶入主文案、標題、CTA、網址  
6. 勾選要投的版本（可只勾一個，有縮圖/封面可辨識）  
7. 必要時在覆寫區改文案；CTA 多數不用改（預設來去逛逛）  
8. 不碰「進階選項」則預設草稿  
9. 點建立  

與 Meta 後台相比：**選完素材包就帶入文案/標題/CTA/網址**，不必在廣告層再貼一次；**CTA 固定下拉、預設來去逛逛**，不用每次選或打；**版本區有縮圖/封面與主版本**，辨識比純檔名快。

### 有沒有重複填素材中心已存在的資訊

- **沒有。** 主文案、標題、CTA、網址皆預設沿用素材包；覆寫區預設摺疊語意為「只有要改才填」。僅在要改時才在覆寫區輸入，否則沿用素材包。

### CTA 是否預設為來去逛逛

- **是。** 表單預設「來去逛逛」；送出時若未填或空白，後端會收到「來去逛逛」。

### 帳號/粉專/IG 是否真的可搜尋下拉

- **廣告帳號：是。** 有同步帳號時為 Popover + Command，可輸入關鍵字篩選名稱或 ID，選完即關閉。  
- **粉專/IG：否。** 仍為選填輸入框，placeholder 說明之後可從下拉選；目前無 API 提供粉專/IG 清單，未實作真正下拉。

### 素材版本區是否真的一眼看得到內容

- **是。** 圖片為 64x64 縮圖；影片為封面圖或 Film 圖示；每列有比例 Badge、類型、主版本標籤；選中列有 ring 與背景色，整列可點選，一眼可辨識要投哪幾支。

---

**結論**：在「選包帶文案、CTA 下拉預設來去逛逛、版本區有預覽與選中狀態、狀態移出主流程、單一尺寸可投」這幾點上，比在 Meta 後台重貼文案、重選 CTA、只看檔名選素材要省步數、辨識較快；粉專/IG 仍待之後接 API 才達到與廣告帳號同級的「可搜尋下拉」。
