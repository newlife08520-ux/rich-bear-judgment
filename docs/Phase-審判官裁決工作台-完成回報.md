# Phase：審判官輸出結構重做（裁決工作台）- 完成回報

## 目標

把審判官從「長文回覆頁」重構成「裁決工作台」，解決難讀、難抓重點、容易疲勞。

## 實際改動

### 檔案

- **`client/src/pages/judgment.tsx`**
  - 新增型別：`ProblemType`、`ParsedJudgment`。
  - 新增常數：`PROBLEM_TYPE_BADGES`、`CONFIDENCE_LABELS`。
  - 新增 `parseJudgmentContent(raw: string): ParsedJudgment`：依 `##`/`###` 標題切段對應到固定欄位；無結構時用啟發式（第一句當總判決、列表當先做什麼、關鍵字匹配問題類型/置信度/是否建議生成任務）。
  - 新增 `JudgmentWorkbenchBubble`：依解析結果渲染固定卡片區塊（頂部摘要卡、詳細原因、具體建議、證據與指標可展開）；支援「一鍵轉任務」。
  - 助理訊息改為使用 `JudgmentWorkbenchBubble`，不再使用 `AssistantBubble`。
  - 新增 `createTaskMutation` 與 `handleCreateTaskFromJudgment`，一鍵轉任務呼叫 `POST /api/workbench/tasks`。

### 必做對應

| 項目 | 狀態 |
|------|------|
| 固定輸出骨架（一句總判決、先做什麼、問題類型、是否建議生成任務、置信度、詳細原因、具體建議、證據與指標可展開） | ✅ 由前端解析並分層顯示 |
| 前端把輸出自動切成卡片區塊 | ✅ 頂部摘要卡 + 詳細原因卡 + 具體建議卡 + 證據卡 |
| 證據與指標改為摘要卡 + 展開細節 | ✅ Collapsible，標題 + 前 80 字摘要，展開後完整內容 |
| 歷史完全移出主閱讀區 | ✅ 維持左側收合側欄，主區僅為對話與裁決卡片 |
| AI 無完整格式時仍盡量抽出總判決／建議動作／關鍵理由 | ✅ 啟發式：第一句、列表、關鍵字匹配 + 無結構時整段 fallback 顯示 |

### 次要

- 問題類型以 Badge 顯示（創意/商品頁/投放/漏斗）。
- 建議動作旁「一鍵轉任務」按鈕，呼叫現有 workbench tasks API。
- 置信度顯示為「高/中/低」Badge。

## 驗收建議

1. **第一眼看到總判決與先做什麼**：進入審判官 → 送出一則判讀 → 確認第一張卡即為「一句總判決」與「先做什麼」。
2. **不需捲完整篇才知重點**：確認頂部摘要卡已涵蓋總判決、問題類型、置信度、先做什麼；詳細原因與具體建議在下方獨立卡。
3. **證據與指標不是一大塊**：若有證據區，應為可收合卡，標題旁僅摘要一行，展開後才見完整。
4. **整體像決策板**：對話區應為卡片式區塊，而非整段長文 prose。

## 備註

- 解析完全在前端（Markdown 標題 + 啟發式）。若後端日後改為回傳結構化欄位（如 JSON），可改為直接使用，前端解析當 fallback。
- 匯出完整對話報告仍使用 `data-print-content`，裁決工作台內之原因/建議/證據區塊均有掛載，列印與匯出可沿用。
