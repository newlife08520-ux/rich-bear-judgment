## 6. 驗收步驟

### 主素材組篩選

1. 素材中心選一素材包，該包內有至少 2 個主素材組（如 A版、B版）且部分版本有歸組、部分未分組。
2. 版本區篩選列選擇「主素材組」= **全部** → 應顯示該包全部版本。
3. 改為 **未分組** → 僅顯示 `groupId` 為空的版本。
4. 改為 **A版** → 僅顯示屬於 A版 的版本。
5. 再搭配「類型」= 圖片、「比例」= 9:16 等 → 結果應為上述篩選後的子集，且排序仍依「最新上傳」或「名稱」正常運作。

### 編輯主素材組

1. 主素材組卡上點某組（如 A版）的 **編輯** → 彈出「編輯主素材組名稱」Dialog，名稱為該組目前名稱。
2. 改為「A版-主推」後儲存 → Dialog 關閉、toast 顯示已更新。
3. 素材中心版本卡上，原屬該組的版本應顯示「A版-主推」。
4. 前往投放中心、選同一素材包 → 批次建組區與快速變體的主素材組列表應顯示「A版-主推」。
5. 檢查既有已建立的投放草稿（若之前用「A版」建立）→ 其 campaign/adSet/ad 名稱應維持建立當下的文字，**不會**被自動改成「A版-主推」。

### 刪除主素材組

1. 主素材組卡上點某組（底下**有**版本）的 **刪除** → 彈出確認框，說明「此主素材組底下仍有版本…」，僅有「關閉」可點，無「刪除」執行。
2. 將該組底下版本改為「未分組」或移到其他組後，再點該組 **刪除** → 確認框為「確定要刪除…」且可點「刪除」。
3. 點「刪除」→ 該組自列表消失，invalidate 後素材中心與投放中心主素材組清單一致。
4. 檢查該包版本列表 → 原屬該組的版本仍存在，且 `groupId` 未被清空或改成其他值（若前端未改歸屬則仍為原 groupId，但該 group 已不存在，屬預期；本輪不實作 B 規則故不自動清空 groupId）。

## 8. 風險與防呆

- **刪除有版本的組**：後端與前端皆檢查；後端 `remove()` 查詢該 package 下 `groupId === 該組` 的版本數，>0 即回傳錯誤；前端在開確認框時也依 `versions` 顯示不同文案，並在 `deleteGroup` 內再次檢查，避免誤刪。防呆：雙重檢查 + 明確提示。
- **編輯組名後投放中心仍顯示舊名**：依賴 React Query 的 invalidate；編輯後僅 invalidate `["/api/asset-packages", selectedPackageId, "groups"]`，投放中心使用同一 query key，下次進入或選該包時會 refetch。若使用者長時間不換頁，可能暫見舊名，屬快取行為，非錯誤。
- **已建立草稿名稱**：本輪未對 PublishDraft 做任何寫入，僅讀取 groups 用於顯示；草稿名稱不會因主素材組改名而變更。
- **版本 groupId 孤兒**：若先刪除某組（底下已無版本），之後才有版本被誤指到該 groupId（例如舊快取或手動改 API），該版本會變成「孤兒 groupId」；本輪不處理此情境，之後若有需要可再補「未分組」或清空無效 groupId 的顯示邏輯。

## 9. 自我檢查後，最可能還殘留的 5 個問題

1. **刪除組後版本仍帶舊 groupId**：採規則 A 不自動清空，若之後有「刪除組時一併把該組版本的 groupId 清空」需求，需另做一輪或選項。
2. **主素材組篩選與其他篩選的順序**：目前為先關鍵字/類型/比例/日期，再主素材組，最後排序；若未來要「先選組再依組內日期篩選」等細需求，可能要調整順序或 UI 說明。
3. **編輯/刪除僅在素材中心**：投放中心不提供編輯或刪除主素材組，若使用者在投放中心想改組名需回素材中心，流程上可接受但未在 UI 提示。
4. **大量主素材組時**：主素材組卡與篩選下拉會變長，目前無分頁或摺疊，若組數很多可能需之後優化。
5. **invalidate 範圍**：編輯/刪除僅 invalidate groups，未 invalidate versions；版本列表的 groupId 來自 versions API，版本卡上的組名來自 `assetGroups.find(g => g.id === v.groupId)?.name`，故 invalidate groups 即會更新顯示，無需再 invalidate versions。

# 額外必答區

## A. 這一輪哪裡開始真正比原本素材整理流程更順

- **原本**：要在一大包版本裡找出「哪些是 A 版、哪些還沒分組」，只能靠檔名或一個一個點開看備註；組名打錯要改只能記住、或刪組重建再重選一次；空組與有版本的組混在一起，刪除時容易誤殺。  
- **現在**：在版本區直接選「主素材組」= 未分組／A版／B版，列表立刻收斂；組名可隨時編輯，改完各處（含投放中心）都會顯示新名稱，不用動已建立的草稿；刪除時系統會擋「底下還有版本的組」，並提示先移出版本或改未分組，**少掉手動對照與誤刪風險**，整理與維護主素材組的成本明顯降低。

## B. 本輪建立／編輯／刪除主素材組的結果明細

- **建立主素材組**：流程與結果與上一輪相同；輸入名稱後 POST groups，列表與投放中心主素材組來源皆為同一 API，無變更。  
- **編輯主素材組**：
