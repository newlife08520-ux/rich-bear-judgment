# START_PHASE.md

以下是本輪開工指令。  
你必須先讀 `AGENTS.md`，再嚴格遵守本輪限制。

---

## 本輪任務
【在這裡寫本輪任務，例如：只做投放中心 Phase E，完成批次建組、快速變體、自動命名、範本、粉專/IG 搜尋下拉】

---

## 本輪允許修改
【在這裡寫允許修改的檔案或模組，例如：
- client/src/pages/publish-placeholder.tsx
- server/modules/publish/*
- server/routes.ts（僅限投放相關 API 掛載）
- shared/schema.ts（僅限投放模板相關欄位）
】

---

## 本輪禁止修改
【在這裡寫本輪禁止修改區域，例如：
- 不准動左側主架構
- 不准動分析區
- 不准動素材中心
- 不准做 Meta 真發送
】

---

## 本輪完成標準
你不能只做到「能跑」。
你必須做到：
1. 功能本身可用
2. UI / API / state 一致
3. loading / empty / error 狀態合理
4. 有明確驗收步驟
5. 有 rollback 方式
6. 有自我檢討
7. 做完就停，不可擅自跳下一階段

---

## 本輪回報格式
做完後必須依 `docs/REPORT_TEMPLATE.md` 完整回報，不可省略。
若有任何一項沒做到，必須明確標示未完成，不可模糊帶過。

---

## 本輪額外要求
【可選填，例如：
- 語氣嚴格
- 自行驗收後再回報
- 若發現規劃偏掉，先自行修正後再回報
- 不要把臨時限制寫成永久規則
】
